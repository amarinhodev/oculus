
## 17:44 -- OCULUS install.py / uninstall.py (cross-platform)
### O que foi feito
Criados `data/oculus/install.py` e `data/oculus/uninstall.py` — scripts de instalação e
desinstalação cross-platform do projeto OCULUS (bot watcher de transcrições Google Meet).

### Fluxo install.py (8 etapas)
1. Detecta OS via platform.system() (Darwin / Linux / Windows)
2. Valida Python >= 3.11 — aborta com mensagem clara se menor
3. Cria venv em ~/.oculus/venv/ e instala requirements.txt
4. Verifica gemini no PATH -> instala via npm se ausente; orienta se npm ausente
5. Copia skills/oculus-analyzer/SKILL.md para o dir de skills do Gemini CLI
6. Configuracao interativa (so no primeiro uso): USER_NAME, OBSIDIAN_VAULT, PREFERRED_LANGUAGE
7. Startup cross-platform:
   - macOS: ~/Library/LaunchAgents/com.oculus.watcher.plist (RunAtLoad+KeepAlive) + launchctl load
   - Linux: ~/.config/systemd/user/oculus.service + systemctl enable/start
   - Windows: XML Task Scheduler com LogonTrigger + schtasks /Create + /Run
8. Mensagem final colorida com link da extensao TranscripTonic

### Fluxo uninstall.py (5 etapas)
1. Detecta OS
2. Para e remove servico (launchctl unload + rm plist / systemctl disable + rm service / schtasks Delete)
3. Remove skill do Gemini CLI (~/.gemini/skills/oculus-analyzer/)
4. Remove venv (~/.oculus/venv/)
5. Pergunta interativa sobre remover config.py (padrao: NAO)

### Decisoes de design
- Ambos os scripts usam apenas stdlib (subprocess, shutil, pathlib, platform, os, sys)
  ate que o venv seja criado — o venv ainda nao existe quando install.py e invocado
- Idempotencia garantida: unload antes de load no plist, delete antes de create no schtasks
- ANSI colors com fallback para Windows (SetConsoleMode via ctypes)
- Cada etapa exibe [check] [!] [x] para feedback visual claro
- Erros criticos abortam com sys.exit(1) + dica de resolucao manual
- Logs em ~/.oculus/ (fora do repositorio — persistencia entre reinstalacoes)

### Artefatos gerados
- data/oculus/install.py
- data/oculus/uninstall.py
